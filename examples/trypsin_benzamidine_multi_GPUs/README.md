# Trypsin-benzamidine (cluster single-GPU and multi-GPU via SLURM)

## System

Same trypsin-benzamidine protein-ligand complex as `trypsin_benzamidine/`, but demonstrating how to run PySTARC on an HPC cluster with SLURM. Two SLURM scripts are provided:

- `submit_SLURM_single_GPU.sh` — runs the entire simulation on one GPU
- `submit_SLURM_multi_GPUs.sh` — splits the simulation across 4 GPUs on one node and combines the results

| Property               | Value                                                   |
|------------------------|---------------------------------------------------------|
| PDB                    | seekrflow manuscript (doi.org/10.1101/2025.08.13.669965)|
| Receptor               | Trypsin (3220 atoms, Q = +6 e)                          |
| Ligand                 | Benzamidine (18 atoms, Q = +1 e)                        |
| b-surface              | 45.0 Angstrom                                           |
| Escape sphere          | 90.0 Angstrom                                           |
| Debye length           | 7.86 Angstrom (150 mM NaCl)                             |
| Reaction pairs         | 10 polar contacts                                       |
| Reaction cutoff        | 6.0-8.5 Angstrom (per pair)                             |
| n_needed               | 4                                                       |
| max_dt                 | 0 (no cap)                                              |
| Born desolvation       | enabled                                                 |
| Hydrodynamic radii     | rec 22.5, lig 5.0 (Angstrom)                            |
| Trajectories           | 10,000,000                                              |
| Experimental k<sub>on</sub> | 2.9 x 10<sup>7</sup> M<sup>-1</sup>s<sup>-1</sup> (Votapka et al. 2022) |

See [`../PARAMETERS.md`](../PARAMETERS.md) Section 2 for parameter selection rationale.

## Prerequisites

Before submitting either script, set `PYSTARC_ROOT` inside the SLURM script to point to the PySTARC installation. Both scripts default to `$HOME/PySTARC`. If the installation is elsewhere, edit the line:

```bash
PYSTARC_ROOT=$HOME/PySTARC
```

Both scripts activate the `PySTARC` conda environment automatically via `conda shell.bash hook`.

## Single-GPU SLURM run

One GPU, 32 CPU threads, 4h walltime. Submits one job that runs setup and the full BD simulation sequentially.

```bash
cd examples/trypsin_benzamidine_multi_GPUs
sbatch submit_SLURM_single_GPU.sh
```

Resource request:

| Resource       | Value            |
|----------------|------------------|
| Partition      | gpu              |
| GPUs           | 1                |
| CPU threads    | 32               |
| Memory         | 128 GB           |
| Walltime       | 4 hours          |

Output files are written to `bd_sims/` in the example directory.

## Multi-GPU SLURM run

Four GPUs on one node, each running 1/4 of the trajectories in parallel. The `multi_GPU_runs.py` splitter generates four `bd_sims/bd_{1..4}/` subdirectories with symlinked DX files and split random seeds. After all four finish, `combine_data.py` pools the results into a single `bd_sims/combined_results.json`.

```bash
cd examples/trypsin_benzamidine_multi_GPUs
sbatch submit_SLURM_multi_GPUs.sh
```

Resource request:

| Resource       | Value            |
|----------------|------------------|
| Partition      | gpu              |
| GPUs           | 4                |
| Tasks per node | 4                |
| CPU per task   | 8                |
| Memory         | 128 GB           |
| Walltime       | 4 hours          |

Workflow:

1. Cleans previous bd_sims and stale output files.
2. Runs `setup.py` to regenerate PQR, rxns.xml, input.xml.
3. Splits input.xml into 4 chunks via `multi_GPU_runs.py`.
4. Launches 4 parallel BD processes, one per GPU (`CUDA_VISIBLE_DEVICES=0..3`).
5. Waits for all 4 to finish.
6. Runs `combine_data.py` to pool results.

Output structure:

```
bd_sims/
  bd_1/                      GPU 0 results
  bd_2/                      GPU 1 results
  bd_3/                      GPU 2 results
  bd_4/                      GPU 3 results
  combined_results.json      Pooled k_on, P_rxn, CI from all 4 GPUs
```

## Output files

After the simulation completes, all results are written to `bd_sims/`. Single-GPU runs produce the standard file set directly in `bd_sims/`. Multi-GPU runs produce one set of files per `bd_*/` subdirectory plus a pooled `combined_results.json`.

| Output file              | Description                                                                                                                    |
|--------------------------|--------------------------------------------------------------------------------------------------------------------------------|
| `results.json`           | k<sub>on</sub>, P<sub>rxn</sub>, Wilson 95% CI, k<sub>b</sub>, D<sub>rel</sub>, wall time, and GPU info (per sub-run).         |
| `combined_results.json`  | Pooled k<sub>on</sub> and CI across all GPU sub-runs (multi-GPU only).                                                         |
| `convergence.json`       | Convergence analysis: SE, relative SE, Wilson CI, convergence verdict, and trajectory estimates for target precision.          |
| `trajectories.csv`       | Per-trajectory record: number of steps, starting pose, minimum distance reached, and number of returns from the escape sphere. |
| `encounters.csv`         | Binding encounter poses for reacted trajectories: final position, orientation, and contact distances.                          |
| `near_misses.csv`        | Trajectories that approached the reaction surface but escaped.                                                                 |
| `fpt_distribution.csv`   | First-passage times for reacted trajectories.                                                                                  |
| `pose_clusters.csv`      | Clustered binding poses from encounter geometries.                                                                             |
| `paths.npz`              | Full trajectory coordinates sampled at configurable intervals.                                                                 |
| `energetics.npz`         | Per-step energies and forces along trajectories.                                                                               |
| `radial_density.csv`     | Radial probability density as a function of distance from the receptor.                                                        |
| `angular_map.npz`        | Angular occupancy map (theta, phi) on the b-surface.                                                                           |
| `contact_frequency.csv`  | Per-pair contact frequencies for the reaction criterion atom pairs.                                                            |
| `milestone_flux.csv`     | Net flux across radial shells.                                                                                                 |
| `transition_matrix.npz`  | Radial shell-to-shell transition counts.                                                                                       |
| `p_commit.npz`           | Commitment probabilities at each radial shell.                                                                                 |

A timestamped log file (`pystarc_YYYYMMDD_HHMMSS.log`) is also written to `bd_sims/` containing the full simulation output.

## Adjusting trajectory count and walltime

Default is 10,000,000 trajectories across 4 GPUs (2.5M per GPU). To change:

1. Edit `setup.py` and modify `N_TRAJECTORIES`.
2. Adjust `--time=4:00:00` in the SLURM script based on expected walltime.
3. For longer runs, increase to `--time=12:00:00` or `--time=24:00:00`.

## Notes

- The multi-GPU splitter generates independent random seeds for each GPU so the four sub-simulations are statistically independent.
- The `bd_sims/bd_*/input.xml` files reference the parent PQR and DX files rather than copying them, saving disk space.
- For interactive debugging, run one GPU directly: `CUDA_VISIBLE_DEVICES=0 python ../../run_pystarc.py input.xml`.
- `PYSTARC_ROOT` must be set correctly inside the SLURM script before submission. The default `$HOME/PySTARC` assumes PySTARC is cloned to your home directory.
